## Scope

In scope:

- pnpm workspace monorepo.
- Docker compose for Postgres and Redis.
- NestJS backend application with env validation, health endpoint, global response envelope, global exception filter, validation pipe, CORS, and Swagger/OpenAPI.
- Next.js App Router frontend shell with original MatchFlow Arena visual direction.
- React Query provider and SDK wrapper placeholder.
- `libs/client_sdk` placeholder and SDK generation command.
- `.env.example` files.
- Documentation and command map.
- Implementation report.

Out of scope:

- Auth, user accounts, roles, organizer profiles.
- Tournament APIs and database migrations.
- Registrations, payments, fixtures, scoring, notifications, and analytics.
- Copied Playmatches branding, assets, UI, copy, or screenshots.

## Data Model Impact

- No business tables are implemented in Slice 1.
- Data model planning is documented in `docs/project-brief.md`, `docs/architecture.md`, and the later feature plans.
- Prisma commands may be present for future migration-first slices.

## API Routes

- `GET /health`: returns API health and metadata.
- Swagger UI: `/docs`.
- OpenAPI JSON: `/docs-json`.

## Frontend Routes/Screens

- `/`: public shell and product entry screen.
- Future route-group directories may be created as placeholders only when useful.

## Permissions

- No protected routes in Slice 1.
- Auth/role enforcement begins in Slice 2.

## OpenAPI/SDK Impact

- OpenAPI generated by Swagger from the backend.
- SDK generation command prepared for `libs/client_sdk`.
- Frontend wrapper exists but should not call handwritten backend endpoint strings for future feature routes.

## Background Jobs Impact

- Redis service is provisioned.
- BullMQ is not implemented until a feature needs it.

## Risks

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Dependency installation may be blocked by network restrictions. | Builds cannot run. | Document blocked commands in the implementation report. |
| Swagger/SDK generation needs a running backend. | SDK command may not complete in Slice 1 verification. | Provide command and wrapper; regenerate during API integration when backend is running. |
| Overbuilding business features during foundation. | Scope creep. | Keep business data and routes out of Slice 1. |
